# OutputTable

Reference to an output table (used in component outputs).

## Type Name
`OutputTable` (internal: `WorkflowComponentParamType.OutputTable`)

## Purpose

`OutputTable` is used to define the **output** of a component, not as an input parameter. It represents the table produced by a component that can be connected to other components' inputs.

## Value Format

Output tables don't have a user-specified value - they are generated by the component's execution.

## Usage in Node Definition

```json
{
  "id": "filter-node",
  "type": "generic",
  "data": {
    "name": "native.where",
    "label": "Filter Data",
    "inputs": [
      { "name": "source", "type": "Table" },
      { "name": "expression", "type": "String", "value": "status = 'active'" }
    ],
    "outputs": [
      { "name": "result", "type": "Table" }
    ]
  }
}
```

The `outputs` array defines what the component produces. The output name (e.g., `"result"`) is used in edge connections via `sourceHandle`.

## Edge Connection

To connect this output to another component's input:

```json
{
  "id": "edge-1",
  "source": "filter-node",
  "target": "next-node",
  "sourceHandle": "result",
  "targetHandle": "source"
}
```

- `sourceHandle`: Matches the output `name` (`"result"`)
- `targetHandle`: Matches the input `name` of the target component

## Common Output Names

| Component Type | Typical Output Name |
|----------------|---------------------|
| Most components | `result` |
| Conditional branches | `true`, `false` |
| Split operations | `result1`, `result2` |

## Multiple Outputs

Some components have multiple outputs:

```json
{
  "outputs": [
    { "name": "matches", "type": "Table" },
    { "name": "nonmatches", "type": "Table" }
  ]
}
```

Each output can be connected to different downstream components.

## Related Types
- `Table` (InputTable) - For receiving table inputs
